# هتل مدیا — نصب‌کننده‌ی فارسی تک‌فایل (بدون Docker)

> راهنمای **توسعه‌دهنده** برای ساخت `Setup-HotelMedia.exe`
> سماع رایانه کیش | kishwifi.com

این پوشه یک نصب‌کننده‌ی **تک‌فایل، فارسی و کاملاً آفلاین** می‌سازد که کاربر غیرفنی فقط
با دابل‌کلیک نصبش می‌کند. هیچ پیش‌نیازی (Docker، Node، دیتابیس، ...) لازم نیست — PHP و
MariaDB و سرور realtime همگی داخل خود فایل بسته‌بندی می‌شوند و به‌صورت **سرویس ویندوز**
(با اجرای خودکار هنگام روشن شدن سیستم) نصب می‌شوند.

---

## تجربه‌ی کاربر نهایی

۱. روی `Setup-HotelMedia.exe` دابل‌کلیک می‌کند (تأیید UAC).
۲. صفحه‌ها همه فارسی و راست‌به‌چپ: خوش‌آمد → **نام هتل** → مسیر نصب (پیش‌فرض `C:\HotelMedia`) → نصب → پایان.
۳. در پایان روی دسکتاپ آیکون **«هتل مدیا»** ساخته می‌شود.
۴. مرورگر خودکار صفحه‌ی خوش‌آمدگویی را باز می‌کند که **نام کاربری و رمز** را نشان می‌دهد.
۵. آیکون کنار ساعت ویندوز (Tray): «باز کردن پنل»، «نمایش رمز ورود»، «راه‌اندازی مجدد»، «خروج».

کاربر هیچ‌وقت CMD/PowerShell/کانفیگ نمی‌بیند و چیزی جز نام هتل تایپ نمی‌کند.

---

## ساخت نصب‌کننده (برای توسعه‌دهنده)

### پیش‌نیاز (فقط روی سیستم build)
- ویندوز ۶۴ بیتی
- [Inno Setup 6](https://jrsoftware.org/isdl.php)
- اتصال اینترنت (PHP/MariaDB/NSSM یک‌بار دانلود و cache می‌شوند)
- *(تری‌اپ با `csc.exe` داخلی ویندوز کامپایل می‌شود — نیازی به Visual Studio نیست)*

### روش ساخت
روی **`build.bat`** دابل‌کلیک کنید، یا:

```powershell
.\installer\hotelmedia\build-hotelmedia.ps1 -Version 1.7.2
```

خروجی:
```
installer\hotelmedia\Output\Setup-HotelMedia.exe   (~۶۵ مگابایت)
```

همین یک فایل را به مشتری بدهید.

## اجزای این پوشه

| فایل | کار |
|------|-----|
| `Setup-HotelMedia.iss` | اسکریپت Inno Setup — UI فارسی RTL، صفحه‌ی نام هتل، rollback، uninstaller با سوال «حذف داده‌ها؟» |
| `build-hotelmedia.ps1` | دانلود runtimeها، ساخت آیکون، کامپایل tray، چیدن payload، کامپایل با ISCC |
| `build.bat` | اجرای تک‌کلیک build |
| `tray/HotelMediaTray.cs` | اپ tray فارسی (C# WinForms، تک‌نمونه) |
| `runtime/provision-hm.ps1` | راه‌اندازی پس از نصب: پورت آزاد، `.env`، دیتابیس، سرویس‌ها، فایروال، صفحه‌ی welcome |
| `runtime/uninstall-hm.ps1` | حذف سرویس‌ها/فایروال/tray هنگام uninstall |
| `runtime/Manage-HotelMedia.ps1` | start / stop / restart / status |
| `runtime/my.ini.template` | کانفیگ MariaDB |

> آیکون، `HotelMediaTray.exe` و باینری‌های PHP/MariaDB/NSSM **در git نیستند** — هنگام
> build ساخته/دانلود می‌شوند. فقط سورس‌ها در مخزن‌اند.

## پشت صحنه (چطور کار می‌کند)

نصب‌کننده فایل‌ها را در `C:\HotelMedia` کپی می‌کند و سپس `provision-hm.ps1` را به‌صورت
کاملاً مخفی (بدون هیچ پنجره‌ی CMD) اجرا می‌کند که:

1. یک **پورت آزاد** برای وب پیدا می‌کند (اگر ۸۰ اشغال بود، خودکار پورت دیگری).
2. `.env` را با **نام هتل**، پورت و رمزهای تصادفی می‌سازد.
3. MariaDB را initialize و به‌عنوان سرویس `HotelMedia-MySQL` ثبت می‌کند.
4. دیتابیس و کاربر را می‌سازد و اسکیما را با `public/install.php` ایمپورت می‌کند.
5. وب‌سرور (`HotelMedia-Web`) و realtime (`HotelMedia-WS`) را به‌عنوان سرویس ثبت می‌کند.
6. فایروال را باز می‌کند و صفحه‌ی `welcome.php` را نشان می‌دهد.

**مدیریت خطا:** اگر هر مرحله شکست بخورد، نصب‌کننده به‌طور خودکار پاک‌سازی (rollback) می‌کند،
یک پیام ساده‌ی فارسی نشان می‌دهد و هیچ فایل خرابی باقی نمی‌گذارد. تمام جزئیات فنی در
`C:\HotelMedia\storage\logs\install.log` ذخیره می‌شود (نه روی صفحه).

**حذف:** از Control Panel یا Start Menu؛ یک سوال فارسی می‌پرسد که آیا اطلاعات هتل هم پاک شود.
